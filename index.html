HTML5 concepts

  • Block-Level Elements
    • Render to begin on a new line
    • May contain inline or other block-level elements
    • Roughli Flow Content(HTML5 category)
  • Inline Elements
    • Render on the same line by default
    • May only contain other inline elements
    • Roughly Phrasing Content(HTML5 category)
  • HTML5 replaces these definitions with more complex set of content categories.
  • However, this distinction remains practical because it aligns well with exiting css rules.

Scope chain

  • Everything is executing in Execution Context
  • Function invocation creates a new Execution Context
  • Each Execution Context has:
    • Its own Variable Enviroment
    • Special 'this' object
    • Reference to its Outer Enviroment
  • Global scope does not have an Outer Enviroment as it's the most outer there is

Summary

  • Difining variables - dynamicaly typed
  • Defining functions - creates its own scope (lexical)
  • JS code runs withing an Execution Context
  • Scope chain is used to retrieve variables from Outer Variable Enviroments

Javascript Types

  • Javascript defines 7 built-in types

    • Objects and 6 primitives

  • Object type is a collection of name/value pairs


    Person Object
    firstName: "Yaakov",
    lastName: "Chaikin",
    social: {
           linkedin: "yaakovchaikin",
           twitter: "yaakovchaikin",
           facebook: "CourseraWebDev"
    }

  • Primitive type can contain a single, immutable value

    • Single value, i.e. not an object
    • Immutable means once it's set, it can't be changed
      • Value becomes read only
      • You can create another value based on existing one

    • Primitive Type: Boolean

      • true
      • false

    • Primitive Type: Underfined

      Underfined signifies that no value has over been set

      • Can only have one value: undefined
      • You can set a variable to undefined, but you should NEVER do it
        • Its meaning is that it's never been defined, so defining it to undefined is counter to its core meaning

    • Primitive Type: Null

      Null signifies lack of value

      • As opposed to undefined, which is lack of defenition
      • Can only have one value: null
      • It's ok to explicitly set a varible to null

    • Primitive Type: Number

      Number is only numeric type in Javascript

      • Always represented under the hood as double-precision 64-bit floating point
      • JS does not hae an integer type
        • Integers are subset of doubles instead of a separate data type

    • Primitive Type: String

      String is sequence of characters used to represent text

      • Use either single ore double quotes, i.e., 'text' or "text"

    • Primitive Type: Symbol

      Symbol is new to ES6 Not covered in this class

      • ES6 (released 2015) isn't widely supported or used yet

  • Undefined means variable memmory has been allocated but no value has ever been explicitly set yet

What is JSON?

JavaScript Object Notation
  • Lightweight data-interchange format
    • Simple textual representation of data
  • Easy for humans to read and write
  • Easy for mashines to parse and generate
  • Completely independent of any language

JSON Syntax Rules

  • Subset of Javascript object literal syntax..but
    • Property names must be in double qoutes
    • String values must be in double qoutes
    • That's it!
  • Syntax for everything else is exactly like for objects literal

Common Misconception

  • JSON is NOT a Javascript Object Literal
  • JSON is just a string
  • The syntax of JSON is based on object literal through
  • Need to convert JSON into JS object

converts drom json string to object

var obj = JSON.parse(jsonString);

converts from object to json string

var str = JSON.stringify(obj);
